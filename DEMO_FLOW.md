# 🏥 医患管理平台 - 演示流程脚本 (Demo Script)

本脚本旨在通过一个完整的 **"患者就诊闭环"**，展示系统如何利用 AI 和自动化技术提升诊所效率与患者体验。

## 🎯 演示核心卖点 (Key Value Propositions)

1.  **AI 赋能临床**：自动识别病灶，辅助医患沟通，提升成单率。
2.  **自动化闭环**：诊疗操作触发库存扣减与账单生成，杜绝漏费与库房烂账。
3.  **多端协同**：医生端 (PC) 与 患者端 (H5) 实时无缝同步。
4.  **严肃医疗**：回归医疗本质，专注病历质量与全周期管理。

---

## 🎬 场景一：患者预约与报到 (Waitless Check-in)

### 1. 快速检索与建档

```text
[患者到店] --> <是否老患?>
                   |
         +---------+---------+
         | 是                | 否
         v                   v
[输入手机号搜索]         [新建档案]
         |                   |
         +---------+---------+
                   |
                   v
             [仪表盘视图]
                   |
                   v
             [报到完成]
```

- **操作**：在 `Patients` 页面，点击 `+ New Patient`。

* **话术**："前台接到电话，只需输入手机号，系统自动检索历史记录。如果是新患者，30秒完成建档，risk assessment 自动标记为 'Low'。"

### 2. 仪表盘概览

- **操作**：切回 `Dashboard`。
- **话术**："院长每天上班第一件事，就是看这个仪表盘。今日预约、实收金额、技工单状态一目了然。不再需要找财务要报表。"

## 🎬 场景二：AI 辅助诊疗 (AI-Assisted Consultation)

### 3. 进入患者详情

- **操作**：点击任意患者（如 "林丽丽"）进入详情页。
- **话术**："这是我们的核心——360度患者视图。左侧是基本信息，中间是牙位图，右侧是病历流。"

### 4. 影像与 AI 诊断 (Wow Moment ✨)

```text
医生          影像系统        AI引擎         电子病历
 |               |             |              |
 | 打开X光片预览  |             |              |
 |-------------->|             |              |
 |               |             |              |
 | 点击'AI分析'   |             |              |
 |---------------------------->|              |
 |               |             |              |
 |               | 扫描动画...  |              |
 |               |<------------|              |
 |               |             |              |
 | 显示结果(98%)  |             |              |
 |<----------------------------|              |
 |               |             |              |
 | 接受并生成报告 |             |              |
 |------------------------------------------->|
 |               |             |              |
 | 病历已保存     |             |              |
 |<-------------------------------------------|
```

- **操作**：

1.  点击 `Imaging` 标签页。
2.  打开一张 **X光片** (点击预览)。
3.  点击右上角 `🤖 AI Analyze` 按钮。
4.  **等待 AI 扫描动画**，展示 "Detected Caries #36 (98%)"。
5.  点击 `Accept & Report`。

- **话术**："看，这就是我们的 AI 引擎。它能在 3 秒内识别龋齿和根尖周炎，准确率 98%。这不仅减少了医生的漏诊，最重要的是——**它能让患者直观地看到问题**，极大地建立了信任感。"

## 🎬 场景三：自动化业务闭环 (Business Automation Loop)

### 5. 一键生成病历与账单

```text
[医生接受 AI 诊断]
       |
       v
 [创建临床病历]
       |
       v
 <触发自动化系统>
       |
       +------------------+
       |                  |
       v                  v
 [库存自动扣减]      [账单自动生成]
       |                  |
       +---------+--------+
                 |
                 v
            (弹出通知)
```

- **操作**：

1.  在刚才 AI 生成的报告中，点击 `Accept`。
2.  系统自动创建一条 **Clinical Note** (电子病历)。
3.  **注意观察右上角 Toast 通知**：
    - `Inventory Deducted: Implant Kit x1`
    - `Invoice Generated: $12,000`

- **话术**："请注意刚才发生了什么。当医生确认治疗方案时，后台发生了两件事：第一，**库房**自动扣除了对应的种植体和耗材；第二，**财务**自动生成了待支付账单。**没有人工干预，没有漏费，库存永远准确。**"

### 6. 财务与库存核查

- **操作**：
  1.  切换到 `Finance` 视图，展示刚才生成的账单。
  2.  切换到 `Inventory` 视图，展示耗材数量的变化。
- **话术**："财务和库管员看到的数据是实时同步的。每一分钱、每一个耗材都有据可查。"

## 🎬 场景四：患者端互动 (Patient Engagement)

### 7. 手机端同步展示

- **操作**：
  1.  打开 **Mobile Simulator** (模拟手机端)。
  2.  展示患者首页：看到刚才的预约已经在 `Next Appointment`。
  3.  点击 `Records`，看到刚才医生写的电子病历和 AI 诊断结果。
- **话术**："患者即刻就能在手机上看到自己的病历和影像。这种透明度是传统诊所无法提供的，它极大地增强了患者粘性。"

## � 场景五：医技协作与 3D 审阅 (Digital Lab Collaboration)

### 8. 技工看板与实时进度

- **操作**：
  1.  切换角色为 `Lab Technician` (或切换到 `Lab` 侧边栏)。
  2.  展示 `Lab Kanban`：刚生成的加工单已经出现在 `Received` 列。
  3.  拖拽工单从 `Received` 到 `Designing`。
- **话术**："在技工端，这笔订单已经实时进入生产看板。医技之间不再需要寄送快递或打冗长的确认电话。"

### 9. 3D 设计远程审核

- **操作**：
  1.  回到患者详情页的 `Lab` 标签。
  2.  点击工单上的 `Review 3D` 按钮。
  3.  **展示 3D 修复体模型**，缩放、旋转演示。
  4.  点击 `Approve Design`。
- **话术**："医生可以直接在诊室预览技工所设计的 3D 冠桥。如果边缘线不对，直接发回修改；如果满意，点击批准，工厂的切削机立即开始工作。这就是**全流程数字化**带来的效率革命。"
